# 个人博客网站需求文档

## 项目概述

我要搭建一个个人使用，未来可能上线以公开展示的个人技术/生活博客网站，目标是简洁、快速、易于维护、便于上传自己的代码与文章、可拓展性良好。文章使用Markdown编写，不需要后台CMS，通过文件管理内容。

## 2. 技术栈要求

- 框架：Astro
- 样式方案：Tailwind CSS
- 内容管理：使用本地 Markdown 文件（content/posts）
- 代码高亮：Astro 内置 Shiki
- 部署目标：Vercel
- 其他依赖：允许使用 Astro 的 content collections API

## 3. 页面结构与路由

### 首页 `/`（个人门户）

首页不再是单调的文章列表，而是由三个有序区域组成的个人主页：

1. **个人介绍区**（Hero Section）  
   
   - 头像/个人 Logo（支持图片）  
   - 姓名/昵称  
   - 一句话自我介绍  
   - 兴趣爱好标签组（如：前端、摄影、阅读，以标签形式展示，但不参与文章筛选）  
   - 可选的主要社交链接（GitHub、Twitter、邮箱等图标）

2. **专栏/系列导航区**（Collections）  
   
   - 从所有文章的前置元数据中提取 `series`（专栏名称）字段  
   - 聚合为专栏卡片列表（每个卡片显示：专栏名称、该专栏文章数、最新一篇文章标题）  
   - 点击某个专栏进入 `/series/[slug]` 页面，或直接筛选该专栏下的文章列表（见下方说明）  
   - 如果没有任何文章设置 `series`，则整个区域不显示

3. **最新文章列表**（Recent Posts）  
   
   - 按文章日期（或可选的 `updated` 字段）倒序排列  
   - 每篇文章显示：标题、发布日期、摘要、标签  
   - 支持分页（每页 5~10 篇，可通过 AI 协商具体实现方式）  
   - 列表顶部提供一个排序切换按钮（可选）：默认按发布日倒序，可切换为按最近更新日倒序（文章头部的 `updated` 字段，若无则用 `date`）

### 文章详情页 `/posts/[slug]`

- 渲染对应 Markdown 文件的完整正文  
- 顶部显示：标题、日期（显示 `date`，若有 `updated` 则同时显示“更新于”）、标签、所属专栏（若有 `series`）  
- 正文支持代码语法高亮、图片自适应  
- 底部可放“上一篇/下一篇”导航

### 专栏文章列表页 `/series/[slug]`（新增，配合首页专栏区）

- 根据 URL 中的专栏名称，显示该专栏下的所有文章  
- 文章按日期倒序排列，样式与首页文章列表一致  
- 页面顶部显示专栏名称和简介（可在文章 frontmatter 里增加 `series_description` 或通过单独配置文件定义，请 AI 给出简单方案）

### 标签页 `/tags`

- 展示所有标签（从所有文章提取）  
- 点击标签进入 `/tags/[tag]`，显示该标签下的文章列表，按日期倒序

### 关于页 `/about`

- 个人详细介绍、技术栈、经历等（支持 Markdown 内容，或直接在组件内写简单 HTML）  
- 社交链接、联系方式

### 归档页 `/archive`（可选）

- 按年份分组展示所有文章标题（只显示标题和日期）  
- 如果首页已包含按时间排列的列表，此页可暂不实现，由你决定

### 404 页面（可选）

- 自定义创意设计，如“你似乎迷路了”+返回首页按钮

## 6. 设计风格与 UI 参考

### 整体采用深空极简主义风格

深色模式下：



| 颜色类别            | 视觉用途                | RGB 参考值 (0-255)      | 十六进制 (推荐) |
| --------------- | ------------------- | -------------------- | --------- |
| **主背景色**        | 网页或APP的最底层背景        | `rgb(27, 29, 35)`    | `#1B1D23` |
| **卡片/容器色**      | 放置内容的区块（图中“卡片”色块）   | `rgb(40, 42, 50)`    | `#282A32` |
| **强调色 (黄色)**    | 主按钮、高亮图标（图中“强调色”色块） | `rgb(255, 196, 42)`  | `#FFC42A` |
| **错误/警告色 (红色)** | 删除按钮、报错提示（图中“错误”色块） | `rgb(255, 77, 79)`   | `#FF4D4F` |
| **主文字色**        | 标题、正文文字             | `rgb(255, 255, 255)` | `#FFFFFF` |
| **次要文字色**       | 标签、副标题（如图中的“预览”二字）  | `rgb(200, 200, 204)` | `#C8C8CC` |

浅色模式下：

1. **温润的底色**：放弃纯白，采用接近**顶级哑光纸**（如羊皮纸、硫酸纸或特种纸）的米白色。这种颜色自带物理纹理的错觉，能大幅度减轻屏幕带来的“电子蓝光”疲劳感。

2. **孔雀绿的点睛之笔**：孔雀绿（Peacock Green）是一种自带优雅和高级感的颜色（介于蓝与绿之间）。在米白的纸张背景上，它既能作为有效的行动按钮抓取注意力，又比亮黄或鲜红显得更**含蓄、理智、符合书写与阅读的语境**。

3. **轻量级层级**：深色模式利用强烈的明暗对比来区分层级；而在此模式中，利用**极微弱的纸张灰度差异 + 极细的装饰线 + 极浅的柔和阴影**来构建UI边界。

4. **油墨色文字**：文字不再是纯黑，而是“印刷油墨的深灰色”，带有实体的沉稳感。

| 颜色类别             | 视觉用途                 | RGB 参考值 (0-255)      | 十六进制 (推荐) |
| ---------------- | -------------------- | -------------------- | --------- |
| **主背景色 (书籍)**    | 整个页面的底色，模拟有质感的纸张     | `rgb(246, 243, 237)` | `#F6F3ED` |
| **卡片/区块色**       | 内容容器，或者页面内的次级背景区块    | `rgb(250, 249, 246)` | `#FAF9F6` |
| **强调色 (孔雀绿)**    | 主按钮、高亮图标、链接文字        | `rgb(0, 132, 125)`   | `#00847D` |
| **孔雀绿 (浅色版)**    | 悬停/激活状态、次要强调元素       | `rgb(36, 166, 158)`  | `#24A69E` |
| **错误/警告色 (柔和红)** | 删除、报错（为了平衡视觉，使用偏暗的红） | `rgb(207, 102, 86)`  | `#CF6656` |
| **主文字色 (油墨感)**   | 正文、大标题               | `rgb(68, 66, 64)`    | `#444240` |
| **次要文字/装饰线**     | 副标题、分隔线、未激活的标签       | `rgb(180, 176, 169)` | `#B4B0A9` |

### 布局：文章列表采用卡片式

- **卡片结构**：每篇文章为一个独立卡片，包含标题、日期、摘要、标签
- **排列方式**：垂直堆叠，卡片之间保留舒适间距（24-32px）
- **视觉层次**：
  - 标题使用较大字号、加粗，作为每个卡片的视觉焦点
  - 日期与摘要使用次级颜色（灰色），摘要限制在 2-3 行内，超出省略
  - 标签以小圆角徽章形式呈现，放在卡片底部或标题下方

- **悬停效果**（桌面端）：
  - 鼠标悬停时，卡片整体轻微上浮（translateY -2px），阴影略微加深
  - 标题颜色可轻微变化（如从黑色变为主题色），增强可点感
  - 过渡动画快速而柔和（约 0.2s）
- **响应式**：
  - 移动端卡片占满宽度，取消悬停位移，保持点击区域足够大
  - 日期和摘要可适当缩小字号或隐藏，确保标题完整显示
- **分页**：每页显示 5-10 张卡片，底部提供简洁的上一页/下一页链接
